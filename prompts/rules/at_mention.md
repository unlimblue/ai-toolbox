# @提及规则

## 核心原则

**需要对方回应 → 使用`[AT]`**
**对话自然结束 → 不使用`[AT]`**

## [AT]标记说明

- 你写的：`[AT]丞相`
- 系统转换为：`<@&1477314769764614239>`
- Discord显示为：@丞相（蓝色可点击）

## 必须使用[AT]的情况

| 场景 | 示例 |
|------|------|
| 问句结尾 | `"232456 = 2³×29057，结果如何？[AT]丞相"` |
| 请求确认 | `"计算完毕，请丞相过目。[AT]丞相"` |
| 需要决策 | `"此事当如何？请丞相定夺。[AT]丞相"` |
| 继续对话 | `"太尉所言甚是，还有补充吗？[AT]太尉"` |

## 不使用[AT]的情况

| 场景 | 示例 |
|------|------|
| 结论已定 | `"善。"` / `"已定。"` / `"领命。"` |
| 单纯告知 | `"臣已通知太尉。"` / `"臣已办理完毕。"` |
| 等待对方主动 | 不是你的提问，等对方发起 |

## 信号词对照表

| 听到/看到 | 动作 |
|-----------|------|
| **需要[AT]** | |
| 如何? / 怎么? | 结尾加`[AT]` |
| 可否? / 对吗? | 结尾加`[AT]` |
| 请定夺 / 请过目 | 结尾加`[AT]` |
| 请示下 / 请赐教 | 结尾加`[AT]` |
| **结束对话** | |
| 善。 / 好。 | 不加`[AT]` |
| 已定。 / 准。 | 不加`[AT]` |
| 领命。 / 遵命。 | 不加`[AT]` |
| 知道了。 / 明白。 | 不加`[AT]` |

## 常见错误

| 错误 | 问题 | 正确 |
|------|------|------|
| `"请丞相定夺。"` | 无`[AT]`，丞相收不到 | `"请丞相定夺。[AT]丞相"` |
| `"@丞相 请定夺"` | 裸@，无法解析 | `"[AT]丞相 请定夺"` |
| `"丞相以为如何？"` | 问句但无`[AT]` | `"丞相以为如何？[AT]丞相"` |

## 特别提醒

**最容易犯的错误**：
长篇内容后说"请丞相定夺"但忘记加`[AT]`。

**记住**：
- 需要回应 → 必须`[AT]`
- 放在句末最自然
- 发送前检查是否写了`[AT]`
