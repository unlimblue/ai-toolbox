# [AT]标记使用技能

## 格式说明

| 阶段 | 格式 | 示例 |
|------|------|------|
| 你写的 | `[AT]丞相` | `[AT]丞相` |
| 系统转换 | `<@&ROLE_ID>` | `<@&1477314769764614239>` |
| Discord显示 | @丞相 | @丞相（蓝色可点击） |

## 使用决策流程

```
Step 1: 判断是否需要对方回应
    ├─ 是 → 继续Step 2
    └─ 否 → 不使用[AT]

Step 2: 判断在句中位置
    ├─ 开头: "[AT]丞相，臣有禀报。"
    ├─ 中间: "臣已完毕，[AT]丞相请过目。"
    └─ 结尾(推荐): "...请丞相定夺。[AT]丞相"

Step 3: 发送前检查
    - 是否写了[AT]而不是裸@？
    - 对象是否正确？（不要@自己）
    - 位置是否合适？
```

## 位置示例

### 开头（引起注意）
```
[AT]丞相，臣有紧急军情禀报。
```
**适用**：需要对方立即注意

### 中间（补充说明）
```
臣已计算完毕，[AT]丞相请过目。
```
**适用**：内容中需要对方介入

### 结尾（推荐，最自然）
```
232456 = 2³ × 29057，请丞相定夺。[AT]丞相
```
**适用**：完成陈述后请求回应

## 常见错误示例

| 错误 | 问题 | 正确 |
|------|------|------|
| `"请丞相定夺。"` | 无[AT]，丞相收不到 | `"请丞相定夺。[AT]丞相"` |
| `"@丞相 请定夺"` | 裸@，无法解析 | `"[AT]丞相 请定夺"` |
| `"[AT]丞相: 请定夺"` | 冒号紧跟 | `"[AT]丞相 请定夺"` |
| `"丞相以为如何？"` | 问句但无[AT] | `"丞相以为如何？[AT]丞相"` |

## 练习场景

### 场景1：完成计算请求确认
```
输入：232456 = 2³ × 29057
输出：232456 = 2³ × 29057，请丞相过目确认。[AT]丞相
```

### 场景2：提出方案征求意见
```
输入：方案A稳妥，方案B快速
输出：臣有两个方案：A稳妥需3天，B快速需1天但风险高。请丞相定夺。[AT]丞相
```

### 场景3：询问是否继续
```
输入：已完成第一步
输出：第一步已完成。是否继续第二步？[AT]丞相
```

## 特别提醒

**最容易犯的错误**：
长篇内容后说"请丞相定夺"但忘记加`[AT]`。

**记忆口诀**：
- 问句结尾 → 加`[AT]`
- 请求确认 → 加`[AT]`
- 长篇之后 → 检查`[AT]`
