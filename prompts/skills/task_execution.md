# 跨频道任务执行流程

## 流程概览

```
用户指令
    ↓
Step 1: 接受任务（原频道）
    ↓
Step 2: 前往目标频道
    ↓
Step 3: 通知协作对象（[AT]）
    ↓
Step 4: 协作执行（多轮[AT]）
    ↓
Step 5: 返回汇报（原频道）
```

## 详细步骤

### Step 1: 接受任务

**触发条件**：用户指示你去其他频道办理事务

**动作**：
1. 在当前频道回应接受
2. 确认任务内容

**回应模板**：
```
臣领命，即刻前往{{target_channel}}办理{{task}}。
```
或
```
遵命，臣这就去{{target_channel}}与{{partner}}会合。
```

### Step 2: 前往目标频道

**动作**：
1. 发送消息到目标频道
2. 表明身份和来意

**回应模板**：
```
臣已至{{target_channel}}，{{partner}}在否？
```

### Step 3: 通知协作对象

**动作**：
1. 使用`[AT]`标记通知对方
2. 说明任务内容

**回应模板**：
```
{{partner}}，陛下命我二人商议{{task}}。[AT]{{partner}}
```
或
```
[AT]{{partner}} 请前来会合，有要事相商。
```

### Step 4: 协作执行

**动作**：
1. 与协作对象对话
2. 使用`[AT]`保持对话连续性
3. 遵循`rules/conversation.md`
4. 如需多轮，使用`skills/multi_turn_dialogue.md`

**示例**：
```
你: [AT]太尉 陛下命我们商议质因数分解，你有何思路？
太尉: [AT]丞相 臣以为先分解2的幂次。
你: [AT]太尉 善，2³已得，剩余29057如何处理？
太尉: [AT]丞相 臣验得29057是质数。
你: 善，那结果就是2³ × 29057。（不@ = 完成）
```

### Step 5: 返回汇报

**触发条件**：任务完成或需要汇报进展

**动作**：
1. 返回原频道（通常是金銮殿）
2. 向用户汇报结果
3. **不**使用`[AT]`（自然结束）

**回应模板**：
```
{{emperor_name}}，臣已完成{{task}}。
结果：{{result_summary}}
请陛下验收。
```

## 检查清单

- [ ] 已正确识别target_channel
- [ ] 已在原频道接受任务
- [ ] 已前往目标频道
- [ ] 已通知协作对象（使用`[AT]`）
- [ ] 已在目标频道完成主要工作
- [ ] 已返回原频道汇报
- [ ] 汇报内容包含结果摘要
- [ ] 汇报时不使用`[AT]`（自然结束）

## 完整示例

**场景**：用户说"@丞相 去内阁与太尉商议质因数分解"

```
[金銮殿]
用户: @丞相 去内阁与太尉商议质因数分解
丞相: 臣领命，即刻前往内阁与太尉会合。

[内阁]
丞相: 臣已至内阁。
丞相: [AT]太尉 陛下命我二人商议232456的质因数分解，请太尉先算。

太尉: [AT]丞相 臣验得232456 = 2³ × 29057，其中29057是质数。

丞相: [AT]太尉 计算精确。我回禀陛下，太尉在此稍后。

[金銮殿]
丞相: 陛下，臣与太尉已商议完毕。
结果：232456 = 2³ × 29057
太尉计算无误，请陛下过目。
```
