## 测试用例 3: 持续对话测试（核心）

**目的**: 验证 Bot 相互 @ 能持续多轮

### 场景 3a: 两 Bot 持续对话
1. 在 **金銮殿** 发送: `@丞相 @太尉，去内阁相互问候并讨论方案`

2. **观察内阁频道**:
   - [ ] 丞相发送: `@太尉 问候语`
   - [ ] 太尉回复: `@丞相 回应`
   - [ ] 丞相继续: `@太尉 进一步讨论`
   - [ ] 太尉回复: `@丞相 回应`

### 预期对话流程（至少 3-4 轮）
```
Round 1:
丞相: <@&1478217215936430092>，太尉大人安好。

Round 2:
太尉: <@&1477314769764614239>，丞相大人客气了。

Round 3:
丞相: <@&1478217215936430092>，此事如何？

Round 4:
太尉: <@&1477314769764614239>，我觉得可行。
```

### 验证点
- [ ] 对话持续 **至少 3 轮**
- [ ] 每轮都包含正确的 `@` 格式
- [ ] 双方 conversation_state 保持 active

## 测试用例 5: 对话终止测试

### 步骤 5a: 自然结束
1. 开始对话（同测试用例 3）
2. 进行 2-3 轮后发送: `@丞相 @太尉，就这样定了`

### 预期结果 5a
- [ ] Bot 总结并结束对话
- [ ] 不再继续 @ 对方

### 步骤 5b: 超时结束
1. 开始对话
2. 等待 **10 分钟** 不发送消息

### 预期结果 5b
- [ ] Bot 自动结束对话状态
