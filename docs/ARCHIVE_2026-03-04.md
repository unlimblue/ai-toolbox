## 4. 多 Bot 系统设计摘要

### 核心架构

```
三层架构：
1. Hub Bot (监听层)    - Hub Token 监听所有频道
2. Message Bus (逻辑层) - 消息分发、上下文过滤、跨频道协调
3. Role Bot (执行层)   - 丞相/太尉各自 Token 发送
```

### 频道配置

| 频道 | ID | 角色 |
|------|-----|------|
| 金銮殿 | 1477312823817277681 | 丞相 + 太尉 |
| 内阁 | 1477312823817277682 | 丞相 + 太尉 |
| 兵部 | 1477312823817277683 | 太尉 |

### 技术栈

| 组件 | 选择 |
|------|------|
| AI 模型 | Kimi (kimi-k2-5) |
| Bot 框架 | discord.py |
| 测试 | pytest + pytest-asyncio |
| Token 安全 | 环境变量 + secrets 文件 |

### System Prompt

- **基础 Prompt**: 提取自 OpenClaw 最佳实践
- **角色特性**: 丞相（统筹决策）、太尉（安全执行）

## 5. Token 安全配置

### 环境变量文件

```bash
# 文件位置: ~/.openclaw/secrets/cyber_dynasty_tokens.env
# 权限: chmod 600
# Git: 已添加到 .gitignore

HUB_BOT_TOKEN=xxx
CHENGXIANG_BOT_TOKEN=xxx
TAIWEI_BOT_TOKEN=xxx
KIMI_API_KEY=xxx
```

### 安全原则

1. 绝不硬编码 Token
2. 环境变量注入
3. 文件权限 600
4. 泄露时立即重置
