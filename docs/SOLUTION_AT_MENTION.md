# 问题诊断与解决方案

## 问题描述

太尉完成计算后，**没有在结尾 @丞相**，导致对话中断。

**实际对话**:
```
太尉: [长篇计算]...故 29057 为素数。终案：232456 = 2³ × 29057
丞相大人以为可否？  ← 这里没有 @丞相
```

**期望对话**:
```
太尉: [长篇计算]...故 29057 为素数。终案：232456 = 2³ × 29057
@丞相 请丞相大人过目定夺。  ← 应该有 @
```

---

## 根本原因

**AI 不清楚"何时应该 @ 对方"**

当前 Prompt 只说了：
- "需要协作时主动 @ 对方"
- "使用 `<@&role_id>` 格式"

但 AI 不理解：**我完成了计算，需要丞相确认，这时候应该 @**

---

## 解决方案

### 方案1：强化 System Prompt（推荐）

在 `config/multi_bot.yaml` 中，为**太尉**添加明确的 @ 规则：

```yaml
 custom_instructions: |
   ## ⚠️ 关键规则：使用 @ 保持对话
   
   **完成计算/任务后，必须 @ 丞相请求确认！**
   
   ### 何时必须 @ 丞相
   
   1. **完成任务后需要确认时**
      - 你完成了皇帝交代的计算/分析
      - 需要丞相确认结果是否正确
      - 例："@丞相 臣已计算完毕，请过目定夺。"
   
   2. **需要对方回应时**
      - 你提出了问题需要丞相回答
      - 你需要丞相的指示才能继续
      - 例："@丞相 此事如何定夺？"
   
   3. **结论需要对方确认时**
      - 你提出了方案需要丞相同意
      - 你需要丞相的最终决策
      - 例："@丞相 臣以为可行，大人以为如何？"
   
   ### 正确示例
   
   用户："与丞相商议质因数分解"
   
   你："丞相大人，臣已计算：
   232456 = 2³ × 29057
   @丞相 请大人过目确认。"
   
   ### 错误示例
   
   你："丞相大人，臣已计算：
   232456 = 2³ × 29057
   请大人过目确认。"  ← ❌ 没有 @，丞相收不到
```

### 方案2：User Prompt 增强

在 `role_bot.py` 的决策 Prompt 中，添加明确提醒：

```python
decision_prompt = f"""
...

## ⚠️ 重要提醒

如果你完成了计算或任务，**必须在最后 @ 对方**请求确认！

**正确格式**:
你的计算/分析内容...
@丞相 请丞相大人过目定夺。

**错误格式**:
你的计算/分析内容...
请丞相大人过目定夺。  ← 没有 @，对话会中断！
"""
```

### 方案3：代码层面兜底（备选）

在 `role_bot.py` 的 `_ai_decide` 中，检查 AI 输出：

```python
async def _ai_decide(self, prompt: str) -> List[Dict[str, Any]]:
    actions = await self._call_ai(prompt)
    
    # 如果当前在对话中，且 AI 没有 @ 任何人，提醒 AI
    if self._is_in_conversation() and not self._has_mention(actions):
        # 重新调用 AI，加上提醒
        reminder = "提醒：你似乎忘记 @ 对方了，请重新生成并记得 @"
        actions = await self._call_ai(prompt + reminder)
    
    return actions
```
